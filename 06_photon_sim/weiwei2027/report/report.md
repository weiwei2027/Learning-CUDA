## 一、问题描述

### 1.1 项目背景

本项目实现了一个 CUDA 加速的 X 射线光子传输模拟器，用于移动式头部 CT 成像系统。战场或灾害现场需要快速颅脑损伤检测，通过模拟 X 射线光子穿过头部多层组织并处理可能存在的球形异物（如血块），生成探测器投影图像。

### 1.2 核心挑战

1. **计算密集**: 需要模拟 10⁷ ~ 10⁹ 个光子的传输过程
2. **几何复杂**: 需要处理多层平板和球形异物的边界
3. **随机性**: 蒙特卡洛方法需要高质量的随机数生成
4. **并行竞争**: 多个线程同时写入探测器像素需要原子操作
5. **精度要求**: 物理模型需正确实现自由程采样和衰减计算

## 二、实现方案

### 2.1 系统架构

```
主机端 (CPU)
├── 文件解析: utils.cpp (几何、材料、源参数)
├── 内存管理: 分配/释放主机和设备内存
├── 数据准备: 构建区域数组、材料系数数组
└── 结果保存: 图像文件、性能日志

设备端 (GPU) 
├── 核函数: photonTransportKernel
│   ├── 65,536 线程并行处理光子
│   ├── 逐层计算路径长度，支持球体相交检测
│   ├── 自由程采样判断是否吸收
│   └── 原子操作累加探测器像素
├── 随机数: cuRAND (每线程独立状态)
├── 常量内存: 存储几何区域和材料参数
└── 内存: 全局内存存储探测器图像
```

### 2.2 核心算法

#### 自由程采样
蒙特卡洛方法核心公式：
```
L = -ln(ξ) / μ
```

#### 球体相交检测
使用光线-球体相交算法判断光子是否经过球体区域：
```
判别式: Δ = b² - 4c
相交条件: Δ ≥ 0 且 t_exit > 0
```

### 2.3 CUDA 优化策略

| 优化点 | 实现方式 | 效果 |
|--------|----------|------|
| 并行粒度 | 65,536 线程，grid-stride 循环处理多光子 | 充分占用 GPU |
| 常量内存 | 几何区域和 μ 值缓存到 __constant__ | 减少全局内存访问 |
| 随机数 | cuRAND，每线程独立状态 | 避免状态竞争 |
| 原子操作 | atomicAdd 累加像素 | 保证数据一致性 |
| 多平台支持 | 抽象头文件 + 平台特定实现 | 支持 4 种国产 GPU |

## 六、遇到的问题与解决方案

### 问题1: 非法内存访问
**现象**: CUDA Error: an illegal memory access was encountered  
**原因**: `h_detector.pixels` 未设置为设备指针  
**解决**: 添加 `h_detector.pixels = d_pixels`

### 问题2: 球体区域处理错误
**现象**: 光子全部被吸收，穿透率为 0%  
**原因**: 球体被当作平板处理，路径计算错误  
**解决**: 跳过非平板区域，实现专门的光线-球体相交检测

### 问题3: Point 模式方向计算错误
**现象**: Point + Sphere 产生亮斑而非阴影  
**原因**: 原实现为"面源+平行束"，非真正点源  
**解决**: 固定源点，方向指向探测器随机目标点

### 问题4: 跨平台编译兼容性
**现象**: 不同平台需要不同的编译器和编译选项  
**解决**: 抽象平台接口，使用条件编译和统一 Makefile
